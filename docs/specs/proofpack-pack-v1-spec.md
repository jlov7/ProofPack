# ProofPack Pack Format v1 Specification

Status: Draft for release-gate hardening  
Version: `1.0.0`  
Applies to: `.proofpack` ZIP bundles generated by ProofPack

## 1. Scope

This specification defines the normative data contract for ProofPack bundles and verification semantics.

- Terms **MUST**, **SHOULD**, and **MAY** are normative.
- Verifiers claiming v1 compliance MUST enforce all MUST-level rules.

## 2. Compatibility Contract

1. `manifest.schema_version` and `receipt.signed_block.schema_version` MUST be identical.
2. A v1 verifier MUST accept only known compatible schema versions.
3. Additive non-breaking changes MUST preserve existing required fields and semantics.
4. Breaking changes MUST increment the major format version and provide an explicit migration path.

## 3. Archive Layout

A conforming pack MUST contain:

- `manifest.json`
- `receipt.json`
- `events/events.jsonl`
- `policy/policy.yml`
- `policy/decisions.jsonl`
- `audit/merkle.json`
- `audit/inclusion_proofs/*.json` (one proof per event)

If selective disclosure is used, pack MAY also contain:

- `disclosure/openings.json`

## 4. Normative File Semantics

### 4.1 manifest.json

Required fields:

- `schema_version`: string, format version token
- `run_id`: UUID v4
- `created_at`: RFC3339 timestamp
- `producer.name`: non-empty string
- `producer.version`: semver-like string
- `event_count`: integer, `>= 0`
- `hashes.events_sha256`: lowercase hex SHA-256
- `hashes.receipt_sha256`: lowercase hex SHA-256
- `hashes.policy_sha256`: lowercase hex SHA-256
- `hashes.decisions_sha256`: lowercase hex SHA-256
- `hashes.merkle_sha256`: lowercase hex SHA-256

### 4.2 receipt.json

`receipt.signed_block` MUST include:

- `schema_version`
- `run_id`
- `created_at`
- `producer`
- `merkle_tree.tree_size`
- `merkle_tree.root_hash`
- `policy.policy_sha256`
- `policy.decisions_sha256`
- `artifact.manifest_sha256`

`receipt.signature` MUST include:

- `alg`: `"Ed25519"`
- `public_key`: base64
- `sig`: base64
- `canonicalization`: `"RFC8785"`
- `hash`: `"SHA-256"`

### 4.3 events/events.jsonl

- MUST be UTF-8 JSON Lines.
- Each line MUST parse to a valid event object.
- Event ordering is semantically relevant and MUST be preserved.

### 4.4 policy/policy.yml and policy/decisions.jsonl

- Policy YAML MUST parse into the supported policy schema.
- Decisions JSONL MUST match event count and carry valid decision entries.

### 4.5 audit/merkle.json and inclusion proofs

- Merkle metadata MUST represent the event stream used for receipt signing.
- Inclusion proof directory MUST contain one entry per event ID.

## 5. Cryptographic Requirements

1. `signed_block` MUST be canonicalized with RFC8785 before signing/verifying.
2. Signature verification MUST use Ed25519 with the provided `public_key`.
3. Merkle tree construction MUST follow RFC6962 domain separation semantics.
4. Policy and decisions hashes MUST match receipt commitments.
5. Disclosure openings (if present) MUST verify against payload commitments.

## 6. Verification Outcome Semantics

A pack is `VERIFIED` only if all required checks pass:

1. `manifest.schema`
2. `receipt.signature`
3. `merkle.root`
4. `merkle.inclusion_all`
5. `policy.hash`
6. `disclosure.openings`

Any failed check MUST set overall verification to false and produce a human-readable failure reason.

## 7. Security and Migration Notes

1. Producers SHOULD avoid private-key reuse across unrelated trust domains.
2. Consumers SHOULD pin expected producer keys out-of-band.
3. Future schema versions SHOULD provide machine-readable migration metadata.

## 8. Implementation Notes

Current implementation baseline in this repository is transitional (`0.1.0`).  
This spec defines the release target contract used by `R01` and migration tasks in `R02`.
